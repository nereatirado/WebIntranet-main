Asignación de responsable en la protección de datos (DPO)

  • Supervisar el cumplimiento normativo
    • El DPO tiene la responsabilidad de garantizar que la organización cumpla con las leyes y regulaciones de protección de datos aplicables, como el Reglamento General de Protección de Datos (RGPD) de la Unión Europea u otras leyes de privacidad locales.
  • Asesoramiento y consultoría
    • El DPO debe proporcionar asesoramiento y orientación a la organización, así como a sus empleados, sobre las mejores prácticas en materia de protección de datos y privacidad. Esto incluye brindar recomendaciones sobre políticas internas, procesos y controles para garantizar el cumplimiento normativo.
  • Monitoreo y auditoría
    • El DPO debe llevar a cabo evaluaciones periódicas de la organización para garantizar el cumplimiento de las leyes de protección de datos. Esto puede incluir la revisión de políticas y procedimientos, así como la realización de auditorías internas para identificar posibles riesgos y vulnerabilidades.
  • Gestión de riesgos
    • El DPO debe identificar y evaluar los riesgos asociados con el procesa- miento de datos personales y desarrollar estrategias y medidas para mitigar esos riesgos. Esto implica llevar a cabo evaluaciones de impacto en la protección de datos (EIPD) cuando sea necesario.
  • Gestión de incidentes
    • El DPO es responsable de gestionar y notificar las violaciones de datos personales a las autoridades de protección de datos competentes y, en algunos casos, a los individuos afectados. También debe establecer y mantener registros de incidentes de seguridad y llevar a cabo investigaciones internas cuando sea necesario.
  • Cooperación con autoridades
    • El DPO actúa como punto de contacto con las autoridades de protección de datos y se encarga de cooperar con ellas en asuntos relacionados con el procesamiento de datos personales, incluyendo la respuesta a solicitudes de información y la facilitación de auditorías y revisiones regulatorias.
  • Sensibilización y formación
    • El DPO actúa como punto de contacto con las autoridades de protección de datos y se encarga de cooperar con ellas en asuntos relacionados con el procesamiento de datos personales, incluyendo la respuesta a solicitudes de información y la facilitación de auditorías y revisiones regulatorias.

Responsabilidades

  • Formación regulada
    • Se proporcionará formación continua al personal del GCT-SOC sobre las políticas y procedimientos de protección de datos
  • Evaluaciones periódicas
    • Se realizarán evaluaciones anuales de riesgos de privacidad y seguridad para identificar posibles brechas o vulnerabilidades en el manejo de los datos.
  • Notificaciones pertinentes
    • Se notificará a las autoridades competentes y a los afectados en caso de violaciones de seguridad o pérdida de datos personales, en cumplimiento de las leyes y regulaciones aplicables.

Seguridad de los datos

  • Acceso autorizado
    • Solo el personal autorizado y debidamente entrenado del GCT-SOC deberá tener acceso a los datos recopilados. Se han establecido mecanismos de autenticación, como contraseñas seguras, autenti- cación de dos factores y control de acceso basado en roles, para garantizar que solo las personas autorizadas puedan acceder a los datos. Un claro ejemplo es el uso de DuoMobile (2FA) para acceder al EDR del GCT-SOC.
  • Segregación de funciones
    • Se realizarán pautas y controles que separen las funciones y los privilegios dentro del SOC. Esto significa que cada miembro del personal tiene acceso solo a la información y los recursos necesarios para desempeñar sus responsabilidades específicas. La segregación de funciones ayuda a prevenir el acceso no autorizado o indebido a los datos.
    • Para más información, consulte la documentación sobre procedimientos del GCT-SOC.
  • Encriptación de datos
    • Los datos almacenados, transmitidos o recopilados, en caso de necesidad, dentro del GCT-SOC deben estar encriptados para proteger su confidencialidad e integridad. La encriptación se aplicará mediante algoritmos de encriptación robustos y prácticas de gestión de claves adecuadas. Los archivos que no tengan información sensible deben estar cifrados mediante el algoritmo AES RSA y el algoritmo ECC para aquellos archivos con TLP RED.
    • Para más información, consultar el procedimiento de categorización de archivos según la sensibilidad de la información.
  • Actualizaciones y parches de seguridad
    • Los sistemas y software del GCT-SOC estarán actualizados con los últimos parches de seguridad y actualizaciones. Esto ayudará a abordar vulnerabilidades conocidas y proteger los datos contra exploits y ataques basados en brechas de seguridad.
  • Análisis de malware
    • Los equipos que tratan con la información recopilada de clientes del GCT-SOC deben tener un EDR instalado.
  • Seguridad de red
    • Para realizar gestiones con los clientes del GCT-SOC se realizarán conexiones vía VPN por motivos de privacidad y anonimato
    • También se contará con tecnologías para monitorear la red, asegurando una protección efectiva de los datos que maneja el GCT-SOC.

Gestión Integral de Datos Personales

  • Recopilación y uso de datos personales
    • El GCT-SOC solo recopilará y procesará los datos personales necesarios para llevar a cabo sus funciones y responsabilidades específicas en materia de seguridad.
    • El GCT-SOC mantendrá registros precisos y actualizados de todas las actividades de procesamien- to de datos personales, incluyendo la finalidad, el plazo de conservación y la base legal para el procesamiento.
    • Para más información, consultar la política de recopilación de datos.
  • Retención y eliminación de datos
    • Los datos personales recopilados y procesados por el GCT-SOC se conservarán únicamente durante el tiempo necesario para cumplir con los fines establecidos, a menos que exista una obligación legal o reglamentaria que requiera un período de retención más prolongado.
    • Se establecerán procedimientos para la eliminación segura de los datos personales una vez que hayan cumplido su propósito original o al finalizar el período de retención.
    • Para más información, consultar Politica de retención y eliminación de Datos.
  • Divulgación de datos personales
    • El GCT-SOC solo divulgará datos personales a terceros cuando exista una base legal para hacerlo o cuando sea necesario para cumplir con una obligación contractual o legal.
    • Antes de divulgar datos personales a terceros, se establecerán acuerdos y salvaguardias apropiadas para garantizar que los datos se protejan y utilicen de manera segura y legal.
  • Derechos de los titulares de datos
    • El GCT-SOC facilitará el ejercicio de los derechos de los titulares de datos, incluyendo el acceso, rectificación, supresión y oposición al procesamiento de sus datos personales, de conformidad con las leyes y regulaciones aplicables.
